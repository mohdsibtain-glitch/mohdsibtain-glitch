<summary><b>📊 &nbsp;Retail Sales &amp; Revenue Analytics — Excel Workbook</b></summary>
<br/>

> Part of a **3-tool retail analytics portfolio** (SQL + Python + Excel). This workbook rolls up a **66,000+ row transaction log** into a fully formula-driven KPI dashboard — nothing is hardcoded, every number in the Dashboard sheet traces back to `SUMIFS`/`COUNTIFS` formulas over the raw `Data` sheet.

**Scope:** Jan 2023 – Dec 2024 &nbsp;|&nbsp; 8 stores &nbsp;|&nbsp; 4 regions &nbsp;|&nbsp; 6 categories

| Metric | Value |
|---|---|
| 💰 Total Net Revenue | ₹11.1 Cr |
| 🛒 Total Orders | 66,325 |
| 📦 Units Sold | 94,128 |
| 🧾 Avg Order Value | ₹1,673.67 |
| 📈 Gross Margin | 34.0% |

**What it demonstrates**
- 3-layer architecture: raw `Data` sheet → `Summary` pivot-style tables → `Dashboard` KPI/chart layer
- Heavy use of `SUMIFS`, `COUNTIFS`, and cross-sheet references for a fully auditable, non-hardcoded model
- Region and category-level margin breakdowns (North, South, East, West all tracked independently)
- Month-over-month growth tracking with automated % change formulas

<summary><b>🎬 &nbsp;Netflix Data Analysis — Python + SQL EDA</b></summary>
<br/>

> Exploratory data analysis on Netflix's public catalog of **8,807 titles (1925–2021)**, combining Python for analysis/visualization and SQL for querying, wrapped in a tabbed interactive report (Overview · Content · Insights · Python code · SQL queries).

| Metric | Value |
|---|---|
| 🎞️ Total Titles | 8,807 |
| 🎬 Movies | 6,131 (69.6%) |
| 📺 TV Shows | 2,676 (30.4%) |
| 🌍 Countries Represented | 748 |
| 📅 Peak Content Year | 2019 (1,424 titles) |
| 🏷️ Top Genre | Drama (36%) |

**Key insights surfaced**
- Rapid catalog growth from 429 titles (2015) → 1,424 titles (2019), slowing sharply post-COVID
- USA leads production (36% of catalog); India is the #2 country with 972 titles
- Content skews adult — `TV-MA` is the single most common rating (3,207 titles)
- Movies outnumber TV shows 2:1, suggesting heavy licensing of legacy movie libraries
- Netflix concentrates content drops in **January** and **October**
- 67% of TV shows run for just one season

<details>
<summary><b>🎓 &nbsp;Student Management System — MySQL Database Project</b></summary>
<br/>

> A relational database design covering the full academic lifecycle of a college — departments, students, courses, enrollments, marks, and attendance — built to demonstrate SQL from beginner joins through advanced procedural logic.

**Schema:** 7 tables (`departments`, `students`, `courses`, `enrollments`, `marks`, `attendance`, `attendance_alerts`) with full referential integrity via foreign keys.

**Query progression demonstrated**
- **Beginner:** filtered selects, joins, `GROUP BY` aggregations
- **Intermediate:** attendance % with `CASE`-based status flags, top scorer per course, pass/fail grading logic, students with no enrollments
- **Advanced:** correlated subqueries, `RANK()` / `DENSE_RANK()` window functions, multi-CTE summary reports, a reusable `student_performance` **VIEW**, a `GetReportCard` **stored procedure**, and an `AFTER INSERT` **trigger** that auto-flags attendance shortages
- Indexing strategy (`CREATE INDEX` / `UNIQUE INDEX`) for query optimization on frequently filtered columns
